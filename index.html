Кратко о проекте

Клиент–серверный интернет-магазин handmade-подарков на Go и PostgreSQL.

20–30 сек: что это, для кого, зачем.

Что разработано

Лендинг, каталог, корзина, заказы, кабинет, отзывы, админ-панель. REST API + PostgreSQL.

Для кого

Автоматизирует приём заказов вместо ручной обработки в мессенджерах.

Съедобное & Несъедобное Техническое решение
7 / 10

Самое сложное техническое решение

Гостевая корзина и слияние при авторизации.

Проблема → решение → результат.

Проблема

Покупатель добавляет товары без регистрации. При входе нельзя потерять корзину и получить дубли.

Решение

CartSession + session_id. После логина POST /api/cart/merge объединяет корзины в транзакции PostgreSQL.

Результат

Товары сохраняются между сеансами. После входа — актуальная корзина и оформление заказа.

Показать личный вклад и умение решать нетривиальные задачи cart · middleware · транзакции
